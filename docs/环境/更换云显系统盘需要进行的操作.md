# 更换云显系统盘需要进行的操作 (Environment Recovery Guide)

当云平台更换实例型号（如 A16 切 4090D）或重置系统盘后，请按照以下步骤快速恢复项目运行环境。所有代码和模型数据均已保存在挂载盘 `~/data` 中，此处仅需恢复系统级配置。

## 安装必备系统工具
进入新机器后，首先安装：
```bash
apt-get update && apt-get install tmux aria2 ffmpeg -y
```

说明：
- 本环境可能没有 `sudo` 命令，直接使用 `apt-get ...`。
- `ffmpeg`/`ffprobe` 是音频解码链路的系统依赖，模块 A 解析 `m4a` 时必须可用。

安装后立刻验证：
```bash
ffmpeg -version
ffprobe -version
```

进一步验证项目音频可被解码（以 `wuli.m4a` 为例）：
```bash
ffprobe -v error -show_entries format=filename,format_name,duration -show_streams ~/data/t1/resources/wuli.m4a
```

预期至少包含：
- `codec_name=aac`（音频流存在）
- `format_name=mov,mp4,m4a,3gp,3g2,mj2`


## 配置 `~/.bashrc`（核心：全局自动激活 `t1/.venv`）
使用 `vim ~/.bashrc`（或直接通过编辑器）打开配置文件，在文件**最末尾**追加以下内容。

这段配置的重点是：打开交互式 shell 后，直接激活 `~/data/t1/.venv`，不再按目录判断，避免误用系统 Python 或 conda base 环境。

```bash
# ==========================================
#[T1 项目专属环境配置]
# ==========================================

# 全局自动激活 t1 虚拟环境：进入任意目录都生效
if [[ $- == *i* ]]; then
    t1_venv="$HOME/data/t1/.venv"
    if [[ -f "$t1_venv/bin/activate" ]] && [[ "${VIRTUAL_ENV:-}" != "$t1_venv" ]]; then
        # 解决与默认 Conda 环境的冲突
        if type conda >/dev/null 2>&1; then
            conda deactivate >/dev/null 2>&1 || true
        fi
        # shellcheck disable=SC1091
        source "$t1_venv/bin/activate"
    fi
fi
# ==========================================
```

## 使配置生效并验证
修改完 `~/.bashrc` 后，执行以下命令立即生效：
```bash
source ~/.bashrc
```

在任意目录执行，确认当前 Python 已是 `t1/.venv`：
```bash
cd ~
echo "$VIRTUAL_ENV"
which python
python -V
```

预期结果：
- `VIRTUAL_ENV` 指向 `~/data/t1/.venv`
- `which python` 指向 `~/data/t1/.venv/bin/python`

## 恢复 Python 依赖 (如果需要)
新系统盘可能没有 `uv` 或缺少部分系统级依赖。进入项目目录后，如果运行报错，执行一次环境同步：
```bash
cdt1
pip install uv  # 如果系统没自带的话
uv sync         # 按 uv.lock 对齐所有依赖库
```
