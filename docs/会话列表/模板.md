# 新会话模板（可直接复制使用；请根据文档名称或协作目标修改本标题）

## 项目位置

- 项目根目录：`<PROJECT_ROOT>`（每次开新会话先按当前环境填写；可为本机或云服务器路径）
- 示例：`M:\MyTest\working\music_video_pipeline`
- AGENTS 文件：`<PROJECT_ROOT>\AGENTS.md`
- 源码主目录：`<PROJECT_ROOT>\src`
- 辅助脚本目录：`<PROJECT_ROOT>\scripts`

## 本次协作目标（默认）

- （按本次会话填写）

## 本次会话记录标题（默认）

- 首条记录建议：`## 本次会话记录（YYYY-MM-DD）`
- 后续记录建议：`## 增量会话记录（YYYY-MM-DD）`
- 同一天内补充内容时，优先追加到同一日期章节，不重复创建同日新章节。

## 当前三条主方向

1. 视觉优化：优先攻克模块 B、模块 C（按实际阶段可包含模块 D）。
2. 用户体验：开发 Web 端界面。
3. 降低时耗：持续优化全链路执行时间。

## 会话记录规则（重要）

- 关键记录文件：  
  `<PROJECT_ROOT>\docs\会话列表\方向\文档整理.md`
- 记录内容仅包含“会话关键信息”，不记录具体开发方案。
- 记录时间点粒度不得细于“天”，但开发调试中的时间统计可以详细。
- 只有当我“显式要求你更新会话记录”时，才更新该文件。

## 执行偏好

- 在未明确要求前，不要主动更新“会话记录文件”。
- 修改文档时优先基于代码事实与仓库现状，不基于假设。

---

```md
## 增量会话记录（YYYY-MM-DD）

### 1. 用户诉求
- 用户新增要求：……

### 2. 本次会话实际动作
- 本轮新增动作：……

### 3. 会话结果
- 本轮结论：……
```

## 使用说明

- 上述模板只记录“关键信息与结果”，不记录具体开发方案。
- 会话记录时间点粒度不细于“天”；开发调试中的时间统计可按需要细化。
