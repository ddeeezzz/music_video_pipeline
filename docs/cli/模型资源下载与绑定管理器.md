uv run --no-sync model_assets

说明（仅交互模式）：
`model_assets` 是统一入口，包含 4 个主功能：
1. 同步 LoRA（bypy）
2. 同步 BaseModel（bypy）
3. 重绑 LoRA key
4. 下载模型资源（HF/直链）

---

## 1. 环境变量（镜像站）

下载 Hugging Face 仓库前，建议先设置镜像站环境变量。

Linux:
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

Windows PowerShell:
```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
```

建议将 Linux 命令写入 `~/.bashrc`，避免每次重复设置。

---

## 2. 功能说明

### 2.1 同步 LoRA（bypy）
- 远端目录：`/lora/{15|xl|fl}/{name}`。
- 目录内必须且只能有一个 `.safetensors`，`.txt` 可选。
- 下载落地：`models/lora/<series>/<name>/`。
- 自动更新：`configs/lora_bindings.json`（唯一键：`binding_name + model_series`）。

### 2.2 同步 BaseModel（bypy）
- 远端目录：`/base_model/{15|xl|fl}/{single|diffusers}/{name}`。
- 下载落地：`models/base_model/<series>/<single|diffusers>/<name>/`。
- 自动更新：`configs/base_model_registry.json`。

### 2.3 重绑 LoRA key
- 从 `configs/lora_bindings.json` 读取绑定记录并分段展示。
- 仅更新：`base_model_key/base_model_path/updated_at`。

### 2.4 下载模型资源（HF/直链）
下载子菜单包含 3 个动作：
1. `LoRA 直链下载并绑定`
2. `BaseModel HF 仓库下载`
3. `BaseModel 直链下载`

#### LoRA 直链下载并绑定
- 流程：选择系列 -> 输入 URL -> 输入 binding 名称 -> 选择底模 key -> 下载并写绑定。
- 落地路径：`models/lora/<series>/<binding_name>/`。
- 来源字段：`lora_bindings.json` 中 `remote_dir` 写为 `direct_url:<url>`。

#### BaseModel HF 仓库下载
- 流程：选择系列 -> 输入 repo_id/revision -> 下载到 diffusers 目录 -> 更新注册表。
- 落地路径：`models/base_model/<series>/diffusers/<repo_name>/`。
- 15/xl 有默认 repo；fl 必须手工输入 repo_id。
- 下载实现：`huggingface-cli download --resume-download`。
- XL 默认启用过滤下载策略，避免拉取 full/openvino/示例资源造成体积膨胀。

#### BaseModel 直链下载
- 流程：选择系列 -> 输入 URL -> 输入保存路径（可默认）-> 下载并更新注册表。
- 保存路径必须位于：`models/base_model/<series>/single/` 下。
